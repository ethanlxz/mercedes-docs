Judge Summary

The core value is the coordination layer.

MEME combines planning, fatigue, battery, and preferences into a single context to provide smart stop recommendations instead of disconnected warnings.

01

Central AI Agent

A LangGraph workflow loads the active road-trip route, fatigue score, EV battery state, and preference memory before producing a driver-facing recommendation.

02

Route-Aware Decisions

Food stops, rest stops, attractions, and chargers are searched ahead on the route, then ranked by actual drive impact, safety priority, Google place data, and remembered preferences.

03

Human Confirmation

The system never silently changes the road trip. It presents one clear action, such as Rest there or Charge there, and updates the route only after acceptance.

System Architecture

Here's how it works.

Input

Driver + Vehicle Context

Origin, destination, recommended stops, active route ID, webcam fatigue frames, EV battery level.

API

FastAPI Gateway

/api/road-trip-planner, /api/central-agent, /api/battery, /ws/fatigue/detect.

Reasoning

LangGraph Agent

Loads road-trip state, evaluates risk, searches stop candidates, builds one recommendation.

Services

Models + Maps

DeepSeek, XGBoost, MediaPipe FaceMesh, Google Routes, Places, and Matrix APIs.

Output

Driver Action

Route card, rest modal, charging plan, updated route polyline, preference memory.

Shared state

A road trip registers through /api/central-agent/active-road-trip, so later fatigue and charging requests reason over the same active route.

Lower latency

The agent reuses route context instead of forcing fatigue, charging, and preferences to rebuild road-trip state independently.

Preference memory

The road-trip planner reads central-agent memory before ranking stops, so preferred stop types are boosted and disliked categories are softened across future recommendations.

Extensible design

New vehicle signals can become graph inputs without rewriting the road-trip UI or duplicating routing logic.

Fatigue Detection

Webcam frames become fatigue scores, then route-aware rest-stop prompts.

The browser streams frames to the WebSocket, MediaPipe produces fatigue signals, and the central agent searches safe stops ahead when warning severity is reached.

Core Technical Modules

Four modules work together, but each has a clear responsibility.

Road Trip Planner

Origin and destination to optimized road trip

The road-trip endpoint takes origin and destination, searches useful stops between them, reads preference memory for liked and disliked stop types, uses Google Routes to rank drive impact, and returns a map-ready route with recommended places.

  1. Collect the driver origin and destination from the road-trip panel.
  2. Use preference memory to order stop searches and score food, rest, attraction, and charger candidates.
  3. Render the final road-trip route, waypoints, distance, and recommendation cards.
Central Agent

One decision point for safety and route context

The central agent connects road-trip geometry, fatigue severity, battery need, and preference memory before creating one actionable notification.

  1. Load active road-trip route and user memory.
  2. Route feedback tasks to memory updates, or search route-relative rest stops and chargers.
  3. Send a clear recommendation with accept and dismiss actions.
Battery Predictor

XGBoost estimates EV usage for road-trip distances

Three models predict road-trip battery need, expected weekday distance, and expected weekday usage from the EV dataset. Charging logic uses the prediction to find the right charger point on the active road-trip route.

  1. Load or train model artifacts from 500_ev.xlsx.
  2. Predict required battery for a road-trip distance.
  3. Recommend no charge, destination charge, or mid-trip charge.
Fatigue Monitor

Webcam frames become route-aware rest suggestions

The browser streams webcam frames over a WebSocket. MediaPipe FaceMesh measures eyes, mouth, blinks, and head motion. When fatigue reaches warning level, the central agent finds a safe stop ahead.

  1. Send resized JPEG frames every 140 ms.
  2. Score eye closure, yawns, blink frequency, and head nods.
  3. Trigger a road-trip rest-stop recommendation when score reaches warning level.
